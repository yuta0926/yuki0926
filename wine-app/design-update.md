# ワイン詳細画面 デザイン実装指示

## 目的

ワイン詳細画面は、ワインごとの基本情報・代表画像・在庫状況・価格・管理情報・コメント・入出庫履歴を一画面で確認できる管理画面とする。

単なる閲覧ページではなく、在庫確認・入出庫登録・編集へ素早く移れる業務用詳細画面として実装する。

現在採用している一覧画面のデザインシステムに合わせ、温かみのあるミニマルな管理画面UIとする。

## 前提

- アプリ全体のデザインシステムは `DESIGN.md` に従う。
- アプリ全体の実装方針は `CLAUDE.md` に従う。
- 一覧画面はリスト形式を標準とする。
- 将来的に画像付きカード表示へ切り替えられる可能性がある。
- ワイン画像は、MVPでは `wines.primary_image_url` としてワインテーブルに保持する。
- 複数画像が必要になった段階で、将来的に `wine_images` テーブルへ分離する。
- 現時点の詳細画面では、代表画像1枚を表示できる設計にする。

## デザイン方針

- 背景は温かいアイボリー系。
- カードは白〜アイボリー系の surface 色を使用する。
- Primary color は深いフォレストグリーン。
- 赤ワイン系のバーガンディ、淡いグリーン、淡いベージュを補助色として使う。
- 高級感は、装飾ではなく余白・罫線・角丸・影の控えめさで表現する。
- ECサイトの商品詳細ページのようにはしない。
- あくまで在庫管理アプリの詳細画面として、情報の確認しやすさを優先する。
- ワイン画像は主役にしすぎず、登録済みの代表画像を確認するための情報資産として扱う。

## 画面構成

詳細画面は以下の構成とする。

1. ヘッダー・サイドバー
2. パンくず・戻る導線
3. ワイン名・ステータス・操作ボタン
4. 代表画像カード
5. 基本情報カード
6. 在庫サマリーカード
7. 価格情報カード
8. 管理情報カード
9. コメント・メモカード
10. 入出庫履歴テーブル

DB方針
画像管理

MVPでは wines.primary_image_url を使用する。

wines.primary_image_url

複数画像が必要になった場合は、将来的に wine_images テーブルへ分離する。

将来案：

wine_images
├── id
├── wine_id
├── image_url
├── image_type

この将来拡張を見越して、フロント側では画像表示コンポーネントを WineImageCard として分離しておく。

入出庫履歴

入出庫履歴を表示するため、inventory_transactions テーブルを追加する。

inventory_transactions
├── id
├── wine_id
├── transaction_type
├── quantity
├── from_location
├── to_location
├── note
├── operated_by
├── transaction_at
├── created_at
└── updated_at

wines.quantity は現在庫数として保持する。

inventory_transactions は、在庫数の変更理由を履歴として保持する。
